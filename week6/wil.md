# 프론트 엔드 스터디 6주차
## 1. fetch와 then
만약 API 요청을 보내 json 데이터가 오기까지 5초가 걸린다고 하자. 브라우저가 코드를 윗줄부터 하나씩 해결한다고 하면(동기) 화면을 구성하는데 오랜 시간이 걸리는 불편함이 생긴다. 이때 fetch 함수와 함께 then 메소드를 사용한다. fetch 함수와 만날 때, API를 처리하고 나면(=then) 보고하라는 의미가 then이다. fetch data는 나중에 오더라도 그외 나머지 코드는 실행된다. 

## 2. 커스텀 훅
훅이 사용된 비슷한 코드들을 묶어서 나만의 훅을 만드는 것을 커스텀 훅이라고 한다. 이번 코드에서는 useState문과 useEffect문을 묶어 커스텀훅에 이용하였다. 중복이 되는 내용은 그대로 적어주고 각 컴포넌트마다 다른 값(이번엔 userId)를 파라미터로 받는다. \\
(e.g.) `fetch(`https://jsonplaceholder.typicode.com/users/${userId}/posts`)`

## 3. 회고
수업을 들을 때는 항상 오오 다 이해했어! 하지만 막상 과제로 다시 코드를 짤 때는 이게 무슨 의미였지..? 하고 헤메는 것 같습니다. 수업 들을 당시엔 100% 이해했다 생각하지만 결과적으로 완전히 이해한 것은 한 68%정도.. 라고 생각합니다. \\
부족한 부분은 useEffect와 비동기 내용, JavaScript 문법 등입니다. 프로젝트 내에서도 따로 공부했지만 한 번 글 읽었다고 바로 이해가 가진 않아서 방학이 되면 좀 더 시간을 들여 찬찬히 들여다보고 싶다 생각합니다. \\
확실히 기억나는 부분은 1, 2, 3주차 내용의 대부분, 4주차의 useState입니다. 초반 내용은 가벼워서 기억에 남고, useState는 이후 프로젝트 내에서 추가로 학습을 더 진행했고, 코드를 더 사용해봤기 때문인 것 같습니다. \\
프론트엔드, 이름만 들어보고 관련해서 공부를 해본 적도 없고 관련된 지식도 정말 단 하나도 없었습니다. 그런데 처음 따라 끄적인 코드가 짱구 홈페이지로 나타났을 때 무척이나 신기했던 감정이 아직도 생생합니다. 저는 원래도 눈에 나타나는 결과물이 있는 만들기를 좋아하는데, 기존의 C++을 비롯한 프로그래밍 언어를 배울 때와는 달리 가시적인 성과(페이지)가 보이니까 무척 흥미로운 경험이였습니다. 문법을 더 알려주셨다면 수업이 너무 딥해지거나 현학적으로 흘렀을 것 같은데 딱 흥미를 붙여주여주시는 정도로 알려주셔서 스터디에 부담이 없어서 좋았습니다.\\

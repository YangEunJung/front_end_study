# 프론트 엔드 스터디 3주차
## 1. 간단하게 협업하기
코드의 규모가 커질수록 유지 보수를 위해 비용이나 개발 기간이 늘어난다. 협업하는 사람이 많거나 코드의 규모가 크다면 협업하기가 어려워질 것이다. 이때 HTML을 사용하지 않고 JS로 모든 화면을 동작하게 만든다면, 즉 각 태그를 컴포넌트로서 활용한다면 재사용성이 올라가며 유지보수 및 협업이 간편해진다. 이를 위해 자바스크립트에 HTML 태그를 넣어서 작업이 가능하게끔 자바스크립트 확장팩 및 컴파일러를 활용한다. (a.k.a. JSX, Javascript Syntax eXtaension)

## 2. SPA
SPA란 Single PAge Application 즉, 한 페이지가 하나의 앱처럼 동작함을 의미한다. JS와 JSX, SPA를 사용한 도구들이 시장에 등장하였으며 그 중 React가 현재 압도적인 점유율을 보인다.

## 3. 번외: 파일명
컴포넌트는 함수명이며 파일명은 대문자로 시작해야한다. (e.g.) `Dog.js`

## 4. 번외: export와 import
각 컴포넌트 마지막 줄에 함수 이름이 Me라면 ` export default Me;` 라는 코드를 추가해준다. export default로 다른 파일에서 얼마든지 import가 가능하게끔 해주는 문장이다. 반대로 이를 사용할 때, 예를 들어 App.js가 Me 함수를 사용할 때는 `import Me from (폴더 경로)` 문장을 시작에 추가한다. 

## 5. React
도서관처럼 원하는 기능을 끌어다 쓸 수 있는 라이브러리이다. 정해진 규칙이 없기에 유연하게 사용할 수 있다. 현재 프론트엔드에서 가장 메이져하게 사용되는 것이 React이므로 성능비교, React와 함께 쓸 수 없는지 등 대부분의 의사 결정의 기준은 React가 된다. 

## npm으로 React 시작하기
1) Vs Code 에서 터미널을 켠 후 `npx create-react-app (폴더명)` 입력하기. react에 필요한 파일을 다운한다. 
2) `cd (폴더명)`, `npm start` 입력하기. React로 된 웹 페이지가 실행된다.
3) 이후 App.js 수정 후 저장시 새로고침을 하지 않아도 사이트에 자동으로 반영된다.
